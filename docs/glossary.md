# プロジェクト用語集 (Glossary)

## 概要

このドキュメントは、Retail Sales Forecasting Pipelineで使用する業務用語、技術用語、状態、評価指標を定義します。

**更新日**: 2026-07-03

## ドメイン用語

### 週次売上

**定義**: 1週間単位で集計された売上金額。

**説明**: 本プロジェクトの目的変数。入力CSVでは `Sales`、内部標準データでは `sales` として扱う。

**英語表記**: Weekly Sales

### 外部要因

**定義**: 売上以外で売上変動に影響する可能性がある説明変数。

**説明**: TVCM、印刷媒体、オフライン広告、デジタル広告などを含む。将来予測では、予測期間分の外部要因が必要になる。

**英語表記**: External Regressor / External Feature

### 実績期間

**定義**: 売上実績が存在し、学習・評価に利用できる期間。

**使用例**: `Record_Type=Actual` の行。

**英語表記**: Actual Period

### 予測期間

**定義**: 将来売上を予測する対象期間。

**使用例**: `Record_Type=Forecast` の行。`Sales` は欠損を許容し、外部要因は必須。

**英語表記**: Forecast Period

### 学習期間

**定義**: モデルのパラメータ推定に使用する過去データの期間。

**英語表記**: Training Period

### 検証期間

**定義**: モデル選択やハイパーパラメータ調整に使用する、学習期間より未来の期間。

**英語表記**: Validation Period

### テスト期間

**定義**: 最終的なモデル性能確認に使用する、学習・検証に使わない未来期間。

**英語表記**: Test Period

### Baseline Model

**定義**: 高度なモデルとの比較基準になる単純な予測モデル。

**説明**: Naive Forecast、Seasonal Naive Forecast、Moving Average Forecastを含む。

**関連用語**: Naive Forecast、Seasonal Naive Forecast、Moving Average Forecast

## 技術用語

### CLI

**定義**: コマンドラインから処理を実行するインターフェース。

**本プロジェクトでの用途**: データ検証、学習、評価、予測、レポート生成を再現可能に実行する。

### pandas

**定義**: Pythonの表形式データ処理ライブラリ。

**本プロジェクトでの用途**: CSV読み込み、前処理、特徴量生成、評価結果の集計。

### LightGBM

**定義**: 勾配ブースティング決定木の機械学習ライブラリ。

**本プロジェクトでの用途**: 広告施策などの非線形な売上影響を捉えるモデル候補。

### Prophet

**定義**: トレンドや季節性を扱う時系列予測ライブラリ。

**本プロジェクトでの用途**: 説明しやすい時系列モデル候補。

### SARIMAX

**定義**: 季節性ARIMAに外生変数を組み込める統計的時系列モデル。

**本プロジェクトでの用途**: 季節性と外部要因を含む統計モデル候補。

## 略語・頭字語

### RMSE

**正式名称**: Root Mean Squared Error

**意味**: 誤差の二乗平均平方根。大きな誤差を強く罰する評価指標。

### MAE

**正式名称**: Mean Absolute Error

**意味**: 絶対誤差の平均。予測誤差の平均的な大きさを示す。

### MAPE

**正式名称**: Mean Absolute Percentage Error

**意味**: 実績値に対する絶対誤差率の平均。実績値が0または小さい場合は扱いに注意する。

### sMAPE

**正式名称**: Symmetric Mean Absolute Percentage Error

**意味**: 実績値と予測値の平均的な大きさで正規化した誤差率。

### WAPE

**正式名称**: Weighted Absolute Percentage Error

**意味**: 絶対誤差合計を実績合計で割った指標。売上予測で解釈しやすい。

### Bias

**正式名称**: Forecast Bias

**意味**: 予測値と実績値の差の平均。正の場合は過大予測、負の場合は過小予測の傾向を示す。

### Baseline比改善率

**正式名称**: Baseline Improvement Rate

**意味**: Baseline Modelの評価指標に対して、対象モデルがどの程度改善したかを示す割合。誤差指標では値が大きいほど改善が大きい。

### GPR

**正式名称**: Gross Rating Point

**意味**: TVCMの出稿量や到達量を表す指標。

## アーキテクチャ用語

### レイヤードアーキテクチャ

**定義**: CLI、アプリケーション、ドメイン、インフラのように責務を層で分離する設計。

**本プロジェクトでの適用**: CLIは処理を調整し、モデル、評価、データ入出力は専用モジュールに分ける。

### Model Registry

**定義**: 設定ファイルのモデル名から、対応するモデルクラスを解決する仕組み。

**本プロジェクトでの適用**: `baseline`、`linear_regression`、`lightgbm` などをCLIから指定できるようにする。

### optional dependency

**定義**: 特定機能を使う場合だけ必要になる追加依存ライブラリ。

**本プロジェクトでの適用**: LightGBM、Prophet、SARIMAXなど、環境構築負荷が高いモデル依存を分離するために使う。

### データリーク

**定義**: 予測時点では利用できない未来情報が学習や特徴量に混入すること。

**本プロジェクトでの対策**: 時系列順の分割、未来売上を使わない特徴量生成、walk-forward validationを採用する。

### Feature Freeze

**定義**: 予測対象時点で利用可能な情報だけを特徴量として固定する考え方。

**本プロジェクトでの適用**: ラグ特徴量、移動平均、外部要因は、予測対象週より未来の実績売上を参照しない。

## ステータス・状態

### Record_Type

| 値 | 意味 | `Sales` | 用途 |
|----|------|---------|------|
| `Actual` | 実績行 | 必須 | 学習、検証、評価 |

## データモデル用語

### RawSalesRecord

**定義**: CSVから読み込んだ直後のレコード。

**主要フィールド**:
- `Date`: 週開始日
- `Sales`: 売上
- `TVCM_GPR`: TVCM指標
- `Print_Media`: 印刷媒体費
- `Offline_Ads`: オフライン広告費
- `Digital_Ads`: デジタル広告費
- `Record_Type`: 実績/予測区分

### SalesRecord

**定義**: 内部処理用に標準化された売上レコード。

**主要フィールド**:
- `week_start_date`: 週開始日
- `sales`: 売上
- `tvcm_gpr`: TVCM指標
- `print_media`: 印刷媒体費
- `offline_ads`: オフライン広告費
- `digital_ads`: デジタル広告費
- `record_type`: 実績/予測区分

## エラー・例外

### 設定エラー

**発生条件**: 設定ファイルが存在しない、必須項目がない、モデル名が不正。

**対処方法**: `configs/base.yaml` のパス、キー、モデル名を確認する。

### データ検証エラー

**発生条件**: 必須カラム不足、日付不正、数値変換不可、`Actual` 行の売上欠損、重複週。

**対処方法**: CLIのエラーメッセージに表示されたカラム、行番号、値を修正する。

### 予測不能エラー

**発生条件**: `Forecast` 行に必要な外部要因がない、学習済みモデルがない、予測期間が不足している。

**対処方法**: 予測期間分の外部要因とモデルartifactを確認する。

## 計算・アルゴリズム

### Naive Forecast

**定義**: 前週売上をそのまま次週予測に使うBaseline。

### Seasonal Naive Forecast

**定義**: 前年同週売上を予測値に使うBaseline。

### Moving Average Forecast

**定義**: 過去一定期間の平均売上を予測値に使うBaseline。本プロジェクトでは過去4週平均を初期対象とする。

### walk-forward validation

**定義**: 学習期間を過去、検証期間を未来に置き、時系列順に複数foldで評価する検証方法。

**目的**: 実運用に近い条件でモデル性能を測定する。
